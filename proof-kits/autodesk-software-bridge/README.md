# Autodesk Software Bridge Proof Kit

Public framework for bridging Autodesk and engineering workflows into general software systems.

## Buyer question
Can CAD events, data contracts, job state, identity, validation, evidence, logs, and business-system handoff be bounded before implementation?

## Review boundary
Public sample architecture only. Drawings, models, Vault data, customer systems, credentials, and native runtime receipts remain private.

## Expected decision output
Integration boundary, representative payload contract, job-state model, validation/evidence requirements, ownership map, and first implementation slice.

For scoped software discovery, use https://tsmithcode.ai/software-discovery-diagnostic.
