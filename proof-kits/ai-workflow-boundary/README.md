# AI Workflow Boundary Proof Kit

Public framework for scoping AI around reviewable operating boundaries rather than hype.

## Buyer question
Are allowed inputs, disallowed data, reviewer responsibility, generated-output rules, fallback behavior, logging, and evaluation checks explicit?

## Review boundary
Synthetic examples only. Private prompts, internal data, customer records, credentials, and production agents remain outside the public proof boundary.

## Expected decision output
Task boundary, input policy, human-review checklist, evaluation rubric, fallback path, audit requirements, and adoption gate.

For scoped software discovery, use https://tsmithcode.ai/software-discovery-diagnostic.
