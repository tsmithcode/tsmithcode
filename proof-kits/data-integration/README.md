# Data Integration Readiness Proof Kit

Public framework for finding schema, ownership, validation, and handoff risk before integration work begins.

## Buyer question
Can source and target systems be connected without hiding ownership, quality, or exception risk?

## Review boundary
Map systems, owners, fields, validation rules, null/duplicate checks, exception paths, reconciliation, and source-of-truth decisions. No customer exports, credentials, or private operating data are included.

## Expected decision output
Source/target map, field contract, validation checklist, exception register, reconciliation plan, and first implementation decision.

For scoped software discovery, use https://tsmithcode.ai/software-discovery-diagnostic.
